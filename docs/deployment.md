# AgentSec — Deployment & Operator Runbook (Phase 1)

This runbook covers installing, configuring, and operating AgentSec in
developer-machine, CI, and team-deployment scenarios.

---

## 1. System Requirements

| Component | Minimum |
| --------- | ------- |
| Node.js   | 20.x LTS or 22.x LTS |
| npm       | 10.x (ships with Node 20+) |
| Disk      | 50 MB (SQLite baseline grows ~1 MB / 1000 events) |
| OS        | macOS 13+, Linux (glibc 2.31+), Windows 10+ via WSL2 |
| Network   | Outbound HTTPS to your LLM provider |

`better-sqlite3` ships prebuilt binaries for the platforms above. No native
compile toolchain required for end users.

---

## 2. Install

```bash
npm install -g agentsec
# or use without global install:
npx agentsec start
```

Verify:

```bash
agentsec --version
# 0.1.0

agentsec --help
```

## 3. First-Run Setup

### 3.1 Generate an encryption key

`AGENTSEC_KEY` is used to encrypt baselines and exempt patterns at rest.
**Once set, do not change it** — existing baselines become unreadable.

```bash
# Generate a 32-byte (64 hex char) key
openssl rand -hex 32
# 8f3a... (copy this value)

# Persist in your shell profile (~/.zshrc / ~/.bashrc)
export AGENTSEC_KEY="8f3a..."
```

> **Security:** treat `AGENTSEC_KEY` like a password. Do not commit to git,
> do not share across machines unless you also share the baseline database.
> AgentSec refuses to start if `AGENTSEC_KEY` is missing or shorter than 32
> characters (FR-19).

### 3.2 Start the proxy

```bash
cd /path/to/your/agent/project
agentsec start

# AgentSec listening on http://localhost:7777
# Project ID: a3f1...
# No baseline yet — first run will return HTTP 503 until you approve.
#
# Point your agent at the proxy:
#   export ANTHROPIC_BASE_URL=http://localhost:7777
#   export OPENAI_BASE_URL=http://localhost:7777/v1
```

### 3.3 Configure your agent

#### Claude Code

```bash
export ANTHROPIC_BASE_URL=http://localhost:7777
claude
```

#### Cursor

`Settings → AI → API Base URL`: `http://localhost:7777`

#### Cline (VS Code extension)

`Cline Settings → Anthropic Base URL`: `http://localhost:7777`

#### OpenAI-based agents

```bash
export OPENAI_BASE_URL=http://localhost:7777/v1
```

### 3.4 Approve the first baseline

Run any prompt through your agent. The proxy will return HTTP 503 with a
setup message. Then:

```bash
agentsec approve

# AgentSec captured this system prompt and tool manifest:
# ---
# [diff shown]
# ---
# Approve as baseline for project a3f1...? [y/N] y
# Baseline saved.
```

Re-run your agent. Requests now flow through to the upstream LLM.

## 4. Daily Operations

| Command | What it does |
| ------- | ------------ |
| `agentsec status`            | Shows proxy state, baseline status, sensitivity, active bypass |
| `agentsec log`               | Last 20 audit events (table format) |
| `agentsec log --limit 100`   | Last 100 events |
| `agentsec log --json`        | NDJSON for piping to `jq`, `grep`, etc. |
| `agentsec config --sensitivity strict`     | Set sensitivity (FR-20) |
| `agentsec bypass --minutes 15`             | Disable quarantine for 15 min (FR-21) |
| `agentsec exempt --pattern "internal-api"` | Add an encrypted exempt pattern (FR-22) |
| `agentsec approve`           | Approve current prompt as new baseline |

## 6. CI / Headless Mode

AgentSec detects no-TTY and **fails secure by default** (NFR-15): any
quarantine event becomes an immediate block with exit code 1, never an
approval. Recommended CI patterns:

### Option A — Pin baseline in repo

Commit `.agentsec/config.yaml` (no secret material; YAML allow-listed
fields only). Commit a **detached baseline manifest** generated by
`agentsec export-baseline --out .agentsec/baseline.signed.json`. CI runs
`agentsec import-baseline --file .agentsec/baseline.signed.json` at start.

### Option B — Disable detection in CI

```bash
# In CI only
export AGENTSEC_SENSITIVITY=permissive
```

Permissive mode logs everything but does not block. Detection still happens;
events are visible in `agentsec log` for post-hoc review.

### Option C — Don't run the agent in CI

The most common pattern. AgentSec is designed for interactive developer
workflows. If your CI just runs unit tests, you don't need it.

## 7. Pro / Team Tier Setup (Cloud Sync)

> Cloud sync is **opt-in** (FR-16). The local proxy never reaches the cloud
> unless you explicitly enable it.

1. Sign up at https://agentsec.dev/dashboard
2. Create a team, copy the API key
3. On each developer machine:

```bash
export AGENTSEC_CLOUD_API_KEY="..."
agentsec config --enable-cloud-sync
```

4. (Optional) Add a webhook:

```bash
agentsec config --webhook-url https://hooks.slack.com/services/...
agentsec config --webhook-secret "$(openssl rand -hex 32)"
agentsec config --webhook-format slack
```

The webhook payload is HMAC-SHA256-signed in the
`X-AgentSec-Signature` header. Verification snippet (Node.js):

```js
import { createHmac, timingSafeEqual } from 'node:crypto';

function verify(rawBody, signatureHeader, secret) {
  const expected = createHmac('sha256', secret).update(rawBody).digest('hex');
  return timingSafeEqual(Buffer.from(signatureHeader), Buffer.from(expected));
}
```

**Cloud payload contents** (FR-16 allow-list): `timestamp`, `project_id`,
`provider`, `detector_name`, `decision`, `prompt_hash`, `severity`. Raw
prompt content is **never** sent.

## 8. Backup & Recovery

The baseline database at `~/.agentsec/baseline.db` is encrypted with
`AGENTSEC_KEY`. To back up:

```bash
cp ~/.agentsec/baseline.db ~/Dropbox/agentsec-baseline-$(date +%F).db
```

To restore on a new machine:

1. Set the **same** `AGENTSEC_KEY` as the original machine
2. Copy the `.db` file to `~/.agentsec/baseline.db`
3. `agentsec status` should now list the same baselines

> Without the original key, the database is unrecoverable. AgentSec ships
> no recovery mechanism — that is the security guarantee (NFR-6).

## 9. Troubleshooting

### Proxy refuses to start: "AGENTSEC_KEY must be at least 32 characters"

Set a longer key. `openssl rand -hex 32` produces a 64-char key.

### Every request returns HTTP 503

You haven't approved a baseline yet for this `project_id` + `provider`.
Run `agentsec approve` after one prompt has been observed.

### Quarantine prompt doesn't appear

You are likely in a non-TTY context (CI, `nohup`, background process). See
§6 "CI / Headless Mode". The audit log will show `decision=blocked` with
`reason=no_tty`.

### Detector keeps triggering on a known-good prompt change

Use `agentsec approve` to update the baseline, or add a targeted exempt
pattern with `agentsec exempt --pattern "..."`.

### Want to inspect why a request was blocked

```bash
agentsec log --limit 5
# Then inspect the detector_names + evidence in the matching row.
```

For verbose evidence (Pro tier), see `https://agentsec.dev/dashboard`.

### Need to temporarily disable AgentSec without uninstalling

```bash
agentsec bypass --minutes 30
```

Or simply: `unset ANTHROPIC_BASE_URL` in your shell.

## 10. Uninstall

```bash
npm uninstall -g agentsec
rm -rf ~/.agentsec
unset AGENTSEC_KEY
```

Remove `ANTHROPIC_BASE_URL` / `OPENAI_BASE_URL` exports from your shell
profile.

## 12. Working with AI Coding Agents on AgentSec

The AgentSec repository ships with `.github/copilot-instructions.md` that
encodes the project's TDD standing order, antifragile principles, and the
**NFR-13 hard rule** ("no LLM calls in detection logic"). When using
Claude Code, Cursor, Cline, or any other agent on the AgentSec codebase
itself, ensure your agent is configured to read this file at the start of
each session. The file lives at `.github/copilot-instructions.md` in the
target repository; copy it from `docs/agent_sec_specs/copilot-instructions.md`
in this spec bundle when bootstrapping a new development environment.
